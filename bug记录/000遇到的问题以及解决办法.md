###### 0.3 插件Unreal Insights 虚幻洞察：帧率分析工具

**0. 打开插件**

**Trace Data Filtering**（追踪数据过滤）

**Insights Data Source Filters**（洞察数据源过滤器）

**1. 从 Unreal Editor (编辑器) 内启动**

这是最简单的方法：

- 在编辑器顶部的菜单栏中，点击 **Tools（工具）**。

- 选择 **Unreal Insights** -\> **Run Unreal Insights**。

- 引擎会自动为你编译并启动这个独立程序。

**2. 找到独立的可执行文件 (根目录)**

如果你想在不打开编辑器的情况下启动它，或者在打包后分析游戏，它位于你的引擎安装目录下：

- **路径格式**：你的安装盘符:\\Epic Games\\UE_5.x\\Engine\\Binaries\\Win64\\

- **文件名**：找到 **UnrealInsights.exe**。

**3. 如何使用它来分析你的"蓝图卡顿"？**

启动 Unreal Insights 后，它会弹出一个 **Session Browser（会话浏览器）**。为了让它记录你的游戏数据，请按以下步骤操作：

**第一步：开启追踪 (Trace)**

在编辑器中点击 **Play（播放）** 运行游戏之前，你需要确保数据正在传输：

1.  在编辑器底部工具栏找到 **Trace（追踪）** 状态图标。

2.  确保 **CPU**、**Frame（帧）**、**Bookmark（书签）** 等频道是开启状态。

**第二步：运行游戏并捕获**

1.  运行你的游戏，复现掉帧严重的场景。

2.  在 **Unreal Insights** 的会话浏览器中，你会看到一个"LIVE（直播）"状态的条目，双击打开它。

**第三步：查看分析结果 (Timing Insights)**

1.  你会看到密密麻麻的条状图。每一列代表一帧。

2.  **找"高峰"**：找到那些特别宽（代表耗时长）的帧。

3.  **看细节**：在下方的 **Timing（时间线）** 面板中，展开 **Game Thread（游戏线程）**。

4.  由于你怀疑是蓝图问题，你会看到很多以 Blueprint 或你的事件名命名的色块。色块越长，说明该逻辑执行时间越久。

**4. 专业建议：捕捉蓝图逻辑的参数**

为了在 Unreal Insights 中看到更详细的蓝图函数名，建议在启动游戏时（通过命令行或快捷方式）添加以下后缀： -trace=default,single 或在控制台输入 Trace.Start。
